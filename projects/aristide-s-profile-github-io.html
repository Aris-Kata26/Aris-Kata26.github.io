Overview

This is the portfolio you are browsing now—rebuilt to surface tutor reports, MixItUp filters, Swiper testimonials, and evidence PDFs in one consistent layout.

Problem

My earlier static site couldn’t handle all the labs, certifications, and reports I kept producing. Recruiters also wanted structured navigation (Services, Experts, Testimonials) instead of one long scroll of cards.

Goal

Build a "case file" site that feels like a living report: fast to scan, auditable links, and light enough to host on GitHub Pages without a framework.

What I built

  • Hero + About fly-out with QR codes, dropdown CV selector, and evidence map grid.
  • Projects grid powered by MixItUp filters plus dedicated detail pages (this task).
  • Swiper sliders for Services, Testimonials, and External Experts to keep layouts compact.
  • Automated contact form + status messages via EmailJS and accessibility-focused markup.

What I learned

Case-study storytelling beats buzzwords; every section now links directly to GitHub, reports, or videos so I can prove claims during interviews.

Architecture

Static HTML + CSS (with CSS custom properties), vanilla JS for interactions, and Swiper/MixItUp CDNs. All assets are bundled under assets/ and deployed via GitHub Pages.

Tech Stack

Frontend: HTML, CSS (custom grid system), vanilla JS, Remix Icon
Libraries: Swiper.js, MixItUp, EmailJS

Key Features

  • Style switcher with stored hue + theme so I can present different accent colors live.
  • Evidence-first cards linking to PDFs, GitHub repos, and YouTube demos.
  • Responsive project detail pages with architecture diagrams and validation notes.

Validation / Testing

Manual regression on desktop + mobile to verify Swiper pagination, MixItUp filters, EmailJS form states, and that every CTA scrolls to the right section.
