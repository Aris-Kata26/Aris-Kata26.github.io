DevOps Workshop

← Back to Projects

Overview

I facilitated a student DevOps workshop that walked classmates from Git commits to automated deployments and GitOps-style change control.

Problem

Many of us could write scripts but hadn’t connected CI/CD concepts end-to-end. We also lacked shared lab notes for when pipelines failed mid-demo.

Goal

Build a hands-on workshop that produces a working pipeline, teaches Git branching discipline, and documents the exact steps so others can rerun it.

What I built

  • Agenda + slides covering IaC basics, CI stages, and GitOps pull-request flow.
  • Sample app + pipeline YAML that compiles, tests, and packages a container.
  • GitHub repo with lab guides, troubleshooting steps, and post-workshop notes.
  • Retro board template so attendees captured blockers for the next run.

What I learned

Teaching forced me to simplify GitOps vocabulary; the most helpful artifact ended up being the rollback checklist, not the slides.

Architecture

Source code lives in GitHub, pipelines run in GitHub Actions (self-hosted runner during class), and deployments target a container host with feedback sent back through pull requests.

Tech Stack

Tools: GitHub, GitHub Actions, Docker/Podman, IaC snippets (Terraform/Ansible samples)
Collaboration: ClickUp task board + shared notes in repo

Key Features

  • End-to-end CI pipeline template with lint, unit test, and container build.
  • GitOps workflow: changes land via pull requests and auto-deploy after approval.
  • Ops runbook detailing secrets management, runner health, and rollback path.

Validation / Testing

Each workshop run executes the pipeline live. We capture logs, confirm deployments on the target host, and intentionally break steps to rehearse recovery.
