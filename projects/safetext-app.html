SafeText App

← Back to Projects

Overview

SafeText is a small Python app that lets classmates encrypt snippets before sending them through shared lab machines.

Problem

We often copy configs or credentials during labs. Plaintext notes linger on shared PCs, so I needed a quick encryption helper that doesn’t require admin rights.

Goal

Provide a portable GUI that generates keys, encrypts/decrypts text with Fernet, and saves ciphertext locally without leaving secrets exposed.

What I built

  • Tkinter UI with panels for key creation, encryption, and decryption.
  • Password gate that unlocks stored keys before allowing crypto operations.
  • Clipboard helpers + local file save so users can move encrypted blobs easily.
  • README with usage steps and screenshots for classmates.

What I learned

Security UX matters: most bugs were people pasting the wrong key, so I added clear states/messages before talking about more algorithms.

Architecture

The app separates UI (Tkinter) from the Fernet crypto engine and a thin storage helper that writes encrypted blobs + key files to disk.

Tech Stack

Language: Python 3
Libraries: Tkinter, cryptography (Fernet), pathlib/os for file handling

Key Features

  • Key generator + management prompts.
  • Inline validation + friendly error states.
  • Lightweight portable bundle (no installer) for lab PCs.

Validation / Testing

Manual tests encrypt/decrypt random strings, simulate wrong passwords, and confirm saved ciphertext decrypts correctly after reopening the app.
