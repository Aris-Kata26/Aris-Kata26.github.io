Private Cloud Storage Project

← Back to Projects

Overview

This lab is my "home cloud"—a self-hosted storage service with MFA-protected access and offline backups.

Problem

Relying on consumer sync apps made it hard to prove my files were encrypted or backed up. I wanted a stack I fully controlled for coursework and internship artifacts.

Goal

Deploy a private storage portal with role-based access, encrypted backups, and documented runbooks teachers can audit.

What I built

  • Storage server with web UI + API for uploads/downloads.
  • MFA-capable auth layer tied to user groups (admin, contributor, viewer).
  • Backup pipeline that snapshots data to an external disk + optional cloud bucket.
  • Monitoring sheet logging capacity, checksums, and restore tests.

What I learned

Backups are useless until you rehearse restores; now every sprint I document one restore test so I know the hashes match.

Architecture

Users authenticate through a web UI, interact with the storage service, and scheduled jobs sync encrypted copies to secondary storage for DR.

Tech Stack

Core: Self-hosted storage server + web admin UI
Security: TLS, MFA plugin, role-based access controls

Key Features

  • Per-user quotas + sharing controls for classmates.
  • Automated backups with checksum verification.
  • Audit log exports for tutor review.

Validation / Testing

Manual test matrix covers login, file CRUD, backup/restore, and permission boundaries. Hash comparisons confirm restores equal original uploads.
