Hybrid Storage Lab

← Back to Projects

Overview

I set up a hybrid storage lab that mirrors MinIO buckets running on-prem to Azure Blob so we can demo resilience without enterprise appliances.

Problem

School labs rely on USB drives or single NAS devices—when a disk fails, we lose class work. We needed something we could run locally but still push to cloud for backups.

Goal

Demonstrate a repeatable pattern where on-prem MinIO handles fast writes while Azure Blob stores an off-site copy with lifecycle policies.

What I built

  • MinIO cluster on lab hardware with buckets mapped to student projects.
  • Azure Storage account + scripts to sync object versions + enforce immutability.
  • Access policy matrix mapping tutor/admin/student permissions.
  • Monitoring sheet that logs replication lag, costs, and failure drills.

What I learned

Calling something “hybrid” is easy until you monitor egress—documenting copy frequency + deletion rules became the biggest win of this lab.

Architecture

MinIO buckets replicate to Azure Blob via client-side sync jobs, with RBAC enforced locally and via Azure AD/Shared Access Signatures for cloud reads.

Tech Stack

Storage: MinIO (self-hosted), Azure Blob Storage
Tools: rclone/azcopy, monitoring scripts, draw.io diagrams

Key Features

  • Bucket policies splitting read-only classroom access from admin writes.
  • Lifecycle rules in Azure to tier older backups to Cool storage.
  • Failover drill checklist showing how to promote Azure copy back to on-prem.

Validation / Testing

I script uploads/reads across both ends, delete files to test versioning, and capture replication timestamps + Azure invoices to prove the pattern stays affordable.
