Overview

This was my first GitHub Pages site where I learned how to package labs, certifications, and PDFs in a public, reviewable way.

Problem

Teachers kept asking for one destination with evidence, but I only had scattered Google Drive links. I needed something lightweight enough for GitHub Pages and fast to update between classes.

Goal

Ship a static site that loads quickly, stays readable on school iPads, and links every claim (projects, badges, PDFs) within two clicks.

What I built

  • Single-page portfolio with sections for projects, services, testimonials, and resume.
  • Theme variables + CSS utility classes so I can restyle cards without touching markup.
  • Link hygiene: every certification or report references real PDFs or GitHub repos.
  • Simple JS interactions (nav toggle, style switcher, smooth scrolling).

What I learned

Maintaining a public site forces me to keep evidence updated; stale badges stand out immediately when everything sits side-by-side.

Architecture

Pure static assets hosted on GitHub Pages with a responsive grid, CSS variables for theming, and a pinch of vanilla JS for navigation + toggles.

Tech Stack

Build: HTML5, custom CSS, vanilla JS
Platform: GitHub Pages + GitHub Actions for deployment

Key Features

  • Light/dark theme toggle that persists via localStorage.
  • Portfolio grid + filter-friendly markup for future MixItUp integration.
  • Inline evidence links (PDFs, YouTube, GitHub) so reviewers can audit fast.

Validation / Testing

Manual checks on Chrome, Edge, and mobile to confirm nav behavior, theme persistence, and that every “View Details” link resolves to an asset.
