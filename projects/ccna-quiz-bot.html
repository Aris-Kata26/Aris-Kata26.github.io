CCNA Quiz Bot

← Back to Projects

Overview

I built a CCNA quiz bot so I could drill subnets, STP, and security topics without relying on commercial exam engines.

Problem

Our class only had short lab blocks, and printed worksheets didn’t give instant feedback. I needed a low-cost way to quiz myself offline and see which domains kept failing.

Goal

Deliver a script that loads custom CCNA question banks, tracks per-topic accuracy, and exports results I can share with my tutor.

What I built

  • Question parser that ingests CSV/JSON pools and tags each item by CCNA domain.
  • CLI quiz loop with timers, hints, and explanations after each answer.
  • Score tracker that summarizes accuracy per topic + saves attempts to disk.
  • Optional export to Markdown so I can drop results into tutor reports.

What I learned

Automating study logs showed me how often I misread multiple-choice timers; throttling question difficulty is as important as memorizing commands.

Architecture

The bot is split into question sources, quiz engine, scoring/reporting, and optional GUI hooks. Everything runs locally so I can study without internet.

Tech Stack

Language: Python 3
Libraries: standard library (csv/json, datetime), optional Tkinter front end

Key Features

  • Themes of questions grouped by VLANs, routing, WLAN, security, etc.
  • Review mode that replays missed questions until I pass twice.
  • Score summaries with timestamps so I can measure progress week to week.

Validation / Testing

Unit-style checks verify parsing, scoring math, and that review queues don’t repeat answered questions. I also cross-check random samples against Cisco official answers.
